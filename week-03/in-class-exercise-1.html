1. Pentabilities (10min)

As last week, please discuss the following question with your group. There are no right / wrong answers and you do not necessarily need to converge to an opinion! The important bit is to think about the concepts and to discuss them.

We will start using the Pentabilities system along two more of the five dimensions today: Cooperation and Autonomy and Initiative. To start familiarising yourselves with the concrete behaviors Pentabilities associates with these concepts, discuss the meaning of the behaviours among the group members. Then choose the two that sound most relevant to you.

Cooperation

  • Listens to others
  • Incorporates what others say
  • Encourages peers’ participation
  • Engages in group decision making
  • Facilitates conflict resolution
  • Recognises personal responsibilities and those of others
  • Helps peers selflessly

Autonomy and Initiative

  • Brings up ideas
  • Asks questions when stuck
  • Plays an active role in group decision making
  • Is able to convince other of his/her approaches
  • Works with determination
  • Believes that he/she can initiate changes

Task 2: Stampede (10min)

You might know the greatest lovers of all music history: Tristan and Isolde. For the purposes of this course, they have an appropriately nerdy way of exchanging love letters, namely by exchanging a little Python script painting a heart.

You will do so now in your group. Please work individually for tasks a. to e. — that is, please do not coordinate at all on the tasks. Of course, you should help each other with technical problems.

  1. Please create / go to the repository for your team by clicking on this link.

  2. Clone the repository to your machine.

  3. Open heart.py in VS Code. You do not need to understand any Python at this point.

    Run the file and display the figure:

    • Press Shift+Ctrl+P to open the drop-down command prompt
    • Start typing “Python: Run Current file in Python Interactive Window” and select it as it pops up

  4. Do you notice anything about the text? Each one of you, feel free to fix the contents, change the colours, do anything you like with it.

  5. Once you are done, make a new commit and push it to the central repository changes.

  6. Pushing to the remote repository will not work for some of you. Discuss among you why that is the case. Try to understand the error message that the team members see.

Task 3: Learning about merge conflicts and fixing them (25min)

  • Go to this website and read about merge conflicts.

  • Explain to each other what merge conflicts are. Be sure that everyone in the team now understands the problem.

  • Move forward. Fix the conflicts. Have a look at this video to see how to do this in VS Code:

  • Continue with Task 4 only after every group member has

    1. fixed a merge conflict and
    2. managed to push his or her changes up to the GitHub repository.

Task 5: Changes in different parts of a file (5min)

  • Make sure everybody has a clean repository and is up-to-date with the GitHub repository (git pull, git status)

  • Pick two group members to change heart.py

    • One person adds

      """My first docstring"""

      to the top of heart.py

    • One person adds

      heart_fig.savefig("heart.png")

      to the bottom of heart.py

  • The first person who is ready pushes his / her changes to the central repository

  • The other person turns on screenshare and goes through the same process as in Task 3.

  • Do you see a merge conflict? Why or why not? Discuss!

Task 5: Visualising branches (10min)

  • The command line is not great for visualisations.
  • So let us look for an extension to VS Code
  • How to go about that?
    1. Search the web for what you want to do. Here: “vs code visualize git branches”.
    2. Git Graph for VS Code will pop up.
    3. Check the download statistics whether it is actually used a lot. (> 10.000 unless very exotic tool)
    4. Follow the installation steps you see on the website (copy to clipboard, hit ctrl+p, paste)
  • Use Git Graph to visualize the history of your “Tristan & Isolde” commits.
  • Take a screenshot, the group with the craziest history will be the team of the day!

Task 6: Branches and sensible workflows. (20min)

  • Are merge conflicts a problem caused by Git? Or by your workflow? Could they be avoided by other collaboration tools that do not require working on the exact same copy in real time (e.g., VS Code + Live Share, Overleaf, Google Docs, …)? Discuss in your group.
  • In Task 3, you have already merged two branches. In particular, you have merged the master branch on GitHub into the master branch on your local machine. To gain a deeper understanding of the power of making explicit branches, please have a look here
  • Think about the remaining assignments. What would be a good workflow for collaboration in your group? Think about making productive use of Git and working asynchronously. That is, do not stop at saying ``We will meet on Zoom and use Live Share all the time.’’.
