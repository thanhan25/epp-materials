Commits and the Staging Area

Hans-Martin von Gaudecker

What makes for a good commit?

  • Should be one logical chunk
  • Goal: Retrieve working versions, revert changes
  • # changes typically decrease in the maturity of a project
  • Rule of thumb: Stop when typing a commit message involving “and”

Examples for commit messages

  • “Moved on with data cleaning: ‘gender’, ‘hh_income’, ‘hh_financial_wealth’.”
  • “Consistently use the variable ‘gender’ throughout, discard ‘male’.”
  • “Fix bug in #44 where return statement was missing from cobb_douglas()”
  • “[WIP] Re-write conclusions.”
  • “Add list comprehension and re-write introduction.”

The staging area

  • Why git add + git commit instead of git commit -a ?
  • Pick logical chunks!
  • Use git add (can be very-fine grained) to put together a logical chunk
  • Then commit this in one go.
