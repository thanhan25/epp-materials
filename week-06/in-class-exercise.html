In-class exercise

Please respect the time limits. Of course you do not need to set a timer, but it is important that you make it to the very end.

Task 1: (10min)

  • Explain each another what the git “staging area” is.
  • Find an example for when it is useful.

Task 2: git history (30min)

  • Go to this repo and click on “Fork” in the upper right corner. You will be asked where to fork to. Choose your own account. Once the process is complete, clone your newly created repository to your local machine.

    Note: we had to use forking here for technical reasons. For the scope of this course, there is no need to understand or research what forks are and why you would use them. You can safely ignore this and just follow the instructions.

  • You may not understand list comprehensions yet. You thus want to get rid of them (if you understand them, you may rightfully conclude that the way that a list comprehension is used here is not yet helpful).

    Use git log, git diff, and git checkout to go through the history of the project and search for the last commit that did not use list comprehensions. Look up the features of those commands as you need them.

  • Use git reset to revert your repository to this commit.

Task 6: Evaluation of Pentabilities (10min)

  • Please rate all dimensions that you could observe.
  • Remember you do not have to rate every behaviour there is; maybe some were not observable. But do think hard about whether you can say something.
  • Give comments if possible; the more concrete your feedback is, the better.
