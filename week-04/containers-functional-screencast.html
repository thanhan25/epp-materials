List: Workhorse among containers

  • Definition: Mutable sequence of objects.

    • mutable: Can change it after creation
    • sequence: An ordered collection
    • of objects: Items can consist of anything

  • Define by comma-separating elements in square brackets

    nonsense = ["abc", 0.0, True]

Tuple: Immutable version of a list

  • Define by comma-separating elements in parentheses

    immutable_nonsense = ("abc", 0.0, True)

  • Careful: Single needs a comma

    immutable_singleton = (5,)

Set: Mutable collection of hashable items

  • Definition: Same as in mathematics (items unique)

  • hashable items ≈ immutable objects

  • Define by comma-separating elements in curly braces

    >>> unique_nonsense = {"abc", 0.0, True, True}
>>> print(unique_nonsense)
{0.0, True, 'abc'}

  • frozenset is the set's immutable cousin

Dictionary: Mapping type

  • Map a set of keys to corresponding values

  • Example:

    utility_parameters = {"ɣ": 0.5, "λ": 0.3}

  • Minimise the number of objects you have to keep in mind!

Which coding style to choose?

  • Classic: Imperative
  • 1990s-2000s: OOP
  • Modern: Functional

Functional programming constructs

  • Three basic building blocks to functional programming:
    • map(function, iterable)
    • filter(function, iterable)
    • reductions (sum, reduce)
